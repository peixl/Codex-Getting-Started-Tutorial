# Codex 新手教程

[English](README.md)

一个面向非技术业务同事的 Codex Desktop 教程站，帮助用户把一句清楚的话变成可在 Windows 或 macOS 本地运行的小工具。

线上地址：**https://codex.ifq.ai**

## 项目概述

这不是 API 文档，也不是只展示视觉效果的 demo。

它解决的是一个很实际的问题：

> 业务同事不会写代码，也能借助 Codex 做出真正能用的本地桌面工具。

站点把这件事拆成了更自然的 4 步：

1. 先看懂 Codex Desktop 能做什么。
2. 用填空式生成器产出更稳的提示词。
3. 复制到 Codex，让它一步步搭建应用。
4. 从部门案例、场景食谱和 FAQ 学会如何描述真实业务问题。

适用场景包括：财务对账、日报汇总、客服助手、人事入职、物流异常、采购报价、投放复盘、合同跟进、行政登记、产品反馈整理等。

## 用户体验目标

- 首次访问按浏览器语言自动进入 `/zh` 或 `/en`。
- 用户手动切换语言后会记住偏好，而不是下次又跳回默认。
- 站点语言切换后，提示词生成器输出语言同步切换。
- 即使用户直接访问 `/guide`、`/faq` 这类无语言前缀路径，也会自动跳到对应语言路径。
- 全站只围绕“本地桌面工具”展开，不把新手拖进服务器、云部署和移动端发布。
- 先用业务语言解释问题，再谈技术实现。

## 主要内容

### 提示词生成器

- 通过轻量表单直接生成可复制到 Codex 的完整提示词。
- 提供快速模板、浏览器本地历史记录、提示词中英文切换。

### 新手上手

- 从第一次打开 Codex，到打包分享给同事，全程用大白话解释。

### 七天上手路线

- 把学习曲线拆成每天一点点，降低第一次接触的压力。

### 部门案例

- 覆盖财务、运营、客服、人事、物流、采购、市场、法务、数据、行政、产品等部门的真实场景。

### 场景食谱 / 进阶技巧 / FAQ

- 帮助用户从“第一次跑通”走到“能稳定写出好提示词”。

## 本地开发

```bash
npm install
npm run dev
```

然后打开 `http://localhost:3000`。

本地验证建议：

1. 根路径是否会按浏览器语言跳转。
2. 手动切换语言后是否能记住偏好。
3. `/guide` 这类裸路径是否会自动补上语言前缀。
4. 生成器在只填写目标和功能时，是否已经能生成可用提示词。

## 项目结构

```text
src/app/[locale]        按语言拆分的页面
src/components          共享 UI、首页模块、生成器、案例组件
src/data                案例内容与提示词模板数据
src/i18n                语言配置与中英文词典
public                  静态资源、manifest、llms.txt、agents.md
```

## 部署

```bash
npm run deploy
```

项目基于 Next.js App Router，通过 `@opennextjs/cloudflare` 部署到 Cloudflare Workers。

## 技术栈

- Next.js 15
- React 19
- TypeScript
- Tailwind CSS 3.4
- Cloudflare Workers
- 基于路径的多语言路由（`/zh/*`、`/en/*`）
