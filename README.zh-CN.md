# Codex 上手指南

[English](README.md)

一个给懂业务的你准备的 Codex Desktop 上手指南，帮助你把清楚的业务流程变成可在 Windows 或 macOS 本地运行的小工具，也学会把 Codex 当成 Agent 处理实际任务。

线上地址：**https://codex.ifq.ai**

## 概述

这不是 API 文档，也不是只展示视觉效果的 demo。

它解决的是一个很实际的问题：

> 把业务做成工具，和 Codex 说清楚就够了。

站点把这件事拆成了更自然的 4 步：

1. 先看懂 Codex Desktop 能做什么。
2. 用填空式生成器产出更稳的提示词。
3. 复制到 Codex，让它一步步搭建应用。
4. 从部门案例、Agent 任务案例、场景食谱和 FAQ 学会描述真实业务流程。

适用场景包括：财务对账、平台账单、日报汇总、客服助手、人事入职、物流异常、承运商评分、采购报价、包装耗材、投放复盘、直播排品、合同跟进、行政登记、产品反馈整理、经营日报、异常订单、客服归因、新品资料、库存补货、采购比价、达人投放、会员分层、优惠风险、类目复盘和评价洞察等。

## 用户体验目标

- 首次访问按浏览器语言自动进入 `/zh` 或 `/en`。
- 用户手动切换语言后会记住偏好，而不是下次又跳回默认。
- 站点语言切换后，提示词生成器输出语言同步切换。
- 即使用户直接访问 `/guide`、`/faq` 这类无语言前缀路径，也会自动跳到对应语言路径。
- 核心应用路线仍围绕“本地桌面工具”展开，同时用 Agent 案例展示更广的 Codex 工作方式。
- 先用业务语言讲场景、流程和验收，再谈技术实现。

## 主要内容

### 提示词生成器

- 通过轻量表单直接生成可复制到 Codex 的完整提示词。
- 提供快速模板、浏览器本地历史记录、提示词中英文切换。

### 快速上手

- 从第一次打开 Codex，到打包分享给同事，只保留必要步骤。

### 七天上手路线

- 把学习曲线拆成每天一点点，降低第一次接触的压力。

### 部门案例

- 覆盖财务、运营、客服、人事、物流、采购、市场、法务、数据、行政、产品等部门的真实场景。

### Agent 任务案例

- 提供可复制提示词，教用户把经营日报、异常订单、客服归因、新品资料、库存补货、采购比价、达人投放、竞品价格、会员分层、优惠风险、直播复盘和类目月报交给 Codex。

### 场景食谱 / 进阶技巧 / FAQ

- 提供订单合并、图片素材包、物流异常、商品标签、平台账单、大促日历、客服话术、达人样品、竞品价格、直播计时、商品巡检和库存调拨等复制即用场景，帮助用户从“第一次跑通”走到“能稳定写出好提示词”。

## 本地运行

```bash
npm install
npm run dev
```

然后打开 `http://localhost:3000`。

本地验证建议：

1. 根路径是否会按浏览器语言跳转。
2. 手动切换语言后是否能记住偏好。
3. `/guide` 这类裸路径是否会自动补上语言前缀。
4. 生成器在只填写目标和功能时，是否已经能生成可用提示词。

## 目录结构

```text
src/app/[locale]        按语言拆分的页面
src/components          共享 UI、首页模块、生成器、案例组件
src/data                案例内容与提示词模板数据
src/i18n                语言配置与中英文词典
public                  静态资源、manifest、llms.txt、agents.md
```

## 部署

```bash
npm run deploy
```

本仓库基于 Next.js App Router，通过 `@opennextjs/cloudflare` 部署到 Cloudflare Workers。

如果你在 Cloudflare 控制台里配置部署，Build command 用 `npm run build` 即可；Deploy command 继续使用 `npx wrangler deploy` 也可以，因为新的构建脚本会先产出 `.open-next` 所需文件。

## 技术栈

- React 19

- Tailwind CSS 3.4

- 基于路径的多语言路由（`/zh/*`、`/en/*`）
